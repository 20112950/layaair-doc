UI. AsynDialog

Le cycle de vie de la boîte de dialogue asynchrone: show ou un popup > oncreate (s'il n'y a pas de créer) > onOpen > onClose > onDestroy (si la destruction) Oncreate dans la page n'est pas créé lors de l'exécution d'une fois, ouvre la page n'est pas exécuté, pour écrire une logique d'exécution uniquement une fois le chargement de ressources, par exemple, un noeud d'auditeur d'événement OnOpen dans chaque ouverture de pages sont exécutées pour faire quelque chose de chaque fois que nécessaire, le message de demande par exemple, en fonction de la page de données d'initialisation OnClose appelé à chaque fois que la fermeture lors de la fermeture, pour arrêter l'animation, la logique de surveillance de messages de réseau Quand onDestroy dans la page détruit l'appel, approprié pour le vide de l'objet de référence
