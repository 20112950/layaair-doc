Core. Sprite

Sprite Class is the basic display object node, through the graphics can draw pictures or vector graphics, support rotation, zoom, displacement and other operations. Sprite is also a container class that is used to add multiple child nodes.

LayaAir engine API design streamlined. The core display class has only one Sprite. Sprite makes rendering optimizations for different situations, so that a class can achieve rich functionality while achieving high performance. Sprite defaults to no width, you can set the width manually, or you can get it by the getbounds function, and you can also set the autoSize=true and then get the width. The width and height of Sprite is only used for collision, and does not affect the display size. If you change the display size, you need to use scaleX, scaleY. Sprite does not accept mouse events by default, that is, mouseEnabled=false, but automatically opens itself and the mouseEnabled=true of all parent objects whenever it is listening to any mouse event. So there's no need to set mouseEnabled manually.

The following sample code creates a <code>Sprite</code> Example。 
<listing version="3.0">
package
{
import laya.display.Sprite;
import laya.events.Event;
public class Sprite_Example
{
private var sprite:Sprite;
private var shape:Sprite
public function Sprite_Example()
{
Laya.init(640, 800);// Set the game canvas wide and render mode. 
Laya.stage.bgColor = "#efefef";// Sets the background color of the canvas. 
onInit();
}
private function onInit():void
{
sprite = new Sprite();// Create a  Sprite  Class instance object  Sprite  . 
sprite.loadImage("resource/ui/bg.png");// Load and display pictures. 
sprite.x = 200;// Set up  Sprite  The horizontal coordinate value of an object relative to the parent container. 
sprite.y = 200;// Set up  Sprite  The vertical coordinate value of the object relative to the parent container. 
sprite.pivotX = 0;// Set up  Sprite  The horizontal method of object; axis point coordinate. 
sprite.pivotY = 0;// Set up  Sprite  Vertical method of object, axis point coordinates. 
Laya.stage.addChild(sprite);// This  Sprite  Object added to display list. 
sprite.on(Event.CLICK, this, onClickSprite);// to  Sprite  Object add click event listening. 
shape = new Sprite();// Create a  Sprite  Class instance object  Sprite  . 
shape.graphics.drawRect(0, 0, 100, 100, "#ccff00", "#ff0000", 2);// Draws a rectangular rectangle with a border. 
shape.x = 400;// Sets the horizontal coordinate value of the shape object relative to the parent container. 
shape.y = 200;// Sets the vertical coordinate value of the shape object relative to the parent container. 
shape.width = 100;// Sets the width of the shape object. 
shape.height = 100;// Sets the height of the shape object. 
shape.pivotX = 50;// Sets the horizontal method, axis point coordinates of the shape object. 
shape.pivotY = 50;// Sets the vertical method, pivot point coordinates of the shape object. 
Laya.stage.addChild(shape);// Add this shape object to the display list. 
shape.on(Event.CLICK, this, onClickShape);// Adds a click event listener to the shape object. 
}
private function onClickSprite():void
{
trace("点击 sprite 对象。");
sprite.rotation += 5;// rotate  Sprite  Object. 
}
private function onClickShape():void
{
trace("点击 shape 对象。");
shape.rotation += 5;// Rotating shape objects. 
}
}
}

alpha :Number

The transparency value is 0-1 and the default value is 1, which means opacity. Changing the alpha value affects drawcall.

autoSize :Boolean

Specifies whether to compute wide and high data automatically. The default value is false. Sprite is wide and high, defaults to 0, and does not change with the rendering content. If you want to get the width of the drawing content, you can set the property as true or get it by the getBounds method. Set to true, has some impact on performance.

blendMode :String

Specifies the blending mode to use. Currently only support "lighter"".

cacheAs :String

Specifies whether the display object is cached as a static image. When cacheAs is changed, the child object is automatically cached, and the reCache method is invoked manually to update the cache.

It is proposed to cache complex content that is not constantly changing into a static image, which can greatly improve rendering performance. CacheAs has "None", "normal" and "bitmap". The three values are optional.
  • The default is "None" and does not do any caching.
  • When the value is "normal", the canvas cache is carried out in canvas mode, and the command cache is in webgl mode.
  • When bitmap is used, canvas mode is still the canvas cache, and the renderTarget buffer is used in webgl mode.
    • Under webgl, the renderTarget cache model has a maximum 2048 size limit, with additional memory overhead and constant overhead when it's redrawn, but it reduces drawcall and renders the highest performance. Under webgl, the command cache mode reduces node traversal and command organization without reducing drawcall performance.
